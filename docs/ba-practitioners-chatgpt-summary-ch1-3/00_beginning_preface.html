Meaning and purpose

How to Use This Study Companion addresses how the business analysis concept supports a better project decision. In the selected scope, it is treated as a practical BA activity that clarifies the business situation, prepares the team for requirements work, or supports investment and planning decisions.

It matters because weak analysis at this point can cause downstream requirements gaps, unclear stakeholder expectations, scope churn, or solution bias. This concept helps the practitioner convert ambiguous input into a decision-ready view.

How to apply it

Apply it by doing four things: define the decision it supports, gather enough evidence, involve the right stakeholders, and document the output in a way that can guide later BA work. Practically, define the concept, connect it to the project context, and document the decision impact.

Practical example: Example: in a software initiative, how to use this study companion may shape which SMEs are interviewed, which acceptance criteria are needed, which risks are escalated, or which option is recommended before development begins.

  • moving to solution design before the business need and decision criteria are stable
  • producing an artifact that no stakeholder uses for a real decision

  4. D. It is only for executives

HardQ4. What is the best use of requirement types?
  1. A. To classify and connect business needs, stakeholder needs, solution behavior, qualities, and transition work
  2. B. To make all requirements technical
  3. C. To avoid stakeholder review
  4. D. To replace traceability

Correct answer: A. Requirement types maintain clarity across different levels of need.

HardQ5. Which planning decision is most likely to reduce requirements rework?
  1. A. Hide the review approach until the end
  2. B. Skip all assumptions
  3. C. Use one template for all projects
  4. D. Define verification, validation, prioritization, and change processes early
Show answer

Correct answer: D. Clear BA planning choices reduce ambiguity and late correction.

HardestQ6. A weighted ranking matrix produces a top option, but the assumptions behind the score are weak. What is the most responsible action?
  1. A. Delete the criteria
  2. B. Avoid discussing risk
  3. C. Approve it because the score is highest
  4. D. Expose the assumptions, test sensitivity, and adjust the recommendation if needed
Show answer

Correct answer: D. Scores support judgment; they do not replace assumption and risk analysis.

HardestQ7. A team says adaptive delivery removes the need for BA planning. Which response is strongest?
  1. A. Planning must be frozen before any learning occurs
  2. B. Adaptive delivery still needs tailored decisions about elicitation, backlog change, validation, and communication
  3. C. Adaptive delivery requires no stakeholders
  4. D. Only predictive projects have requirements
Show answer

Correct answer: B. Adaptive work needs fit-for-purpose planning, not absence of planning.

HardestQ8. Which symptom suggests current-state analysis was too shallow?
  1. A. The team can explain capability gaps with evidence
  2. B. The team discovers during implementation that the real blocker is policy, not the system
  3. C. Stakeholders agree on the situation statement
  4. D. Options are compared against criteria

Correct answer: B. Discovering a basic root cause late suggests insufficient current-state and root-cause work.

HardestQ9. What is the main risk of approving a BA plan without reviewing it with key stakeholders?

  2. B. The glossary grows too long
  3. C. The site navigation fails
  4. D. The source PDF becomes unavailable

Correct answer: A. Review builds ownership and uncovers planning risks.

HardestQ10. Why should the business case remain visible after project approval?

  3. C. It helps evaluate changes and confirm delivery remains aligned with expected value
  4. D. It replaces all test cases
