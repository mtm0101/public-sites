X2.1 Groups of Tools and Techniques (Nhóm Công cụ và Kỹ thuật)

What it is

A way to organize ba techniques by the question they help answer rather than by alphabetical memorization. For section X2.1, treat the material as a practical aid that complements professional judgment rather than replacing it.

Why it matters

For section X2.1, the main value is turning professional knowledge into usable BA work products while improving review discipline and decision confidence.

When to use it

Use this section when choosing how to discover, model, analyze, prioritize, trace, validate, or evaluate product information. Use section X2.1 when the situation contains ambiguity, cross-functional dependency, quality risk, or a need to explain recommendation logic.

How to apply it

Start by naming the decision or capability being supported. For section X2.1, choose the lightest useful evidence, bring in the right reviewers, document assumptions, and produce a reusable artifact.

Practical example

If the question is “what is the real problem,” root cause analysis may fit. If the question is “which option should we choose,” weighted ranking or cost-benefit analysis may fit. If the question is “what changes if this requirement moves,” traceability and impact analysis become more relevant.

Common mistakes

  • Choosing a familiar technique without considering the decision
  • Using a tool because it looks impressive rather than because it answers a question

Quick checklist

  • Start with the question
  • Match the technique to evidence type
  • Record why a technique was chosen

The BA should translate it into targeted questions, diagrams, decision records, backlog notes, acceptance criteria, or improvement actions.

Key takeaway: Groups of Tools and Techniques should make business analysis work more trustworthy, reusable, and decision-oriented.
