REP_MBL001
Mobile App Usage Report (MAUR)
The MAUR contains the total numbers of new app downloads, recipe uploads, and frequency of visits by
app users
The MAUR will be used to determine if the Recipe Box feature is being utilized by mobile app users and if users
are actually purchasing the products in stores
Business Objective 01 (improve profit per visit)
1 of 16
Sales and Marketing
Inventory Report (REP_INV001), Sales Report (REP_REV001)
Marketing Manager
VP of Sales, VP of Marketing, Sales Representatives, Marketing Department, Mobile Services Representatives
MAUR is generated automatically on Monday morning at 2 a.m.
Weekly
MAUR is delivered within 2 minutes of being triggered and contains data for the previous week
(Monday 12:00 a.m.–Sunday 11:59 p.m.)
Approximately 100,000 transactions are logged weekly
Viewable by all sales and marketing team members
All settings are saved between report executions
The Reporting period will be noted above the MAUR Matrix in the form "Month DD, YYYY – Month DD, YYYY"
MAUR is a matrix with the following columns: Number of App Downloads, Number of Recipe Uploads, Store ID,
Region, Number of Loyalty Program Transactions, Number of Loyalty Program Transactions by Users with Recipe
Uploads, Average Number of Products per Transaction for Loyalty with Recipe Transactions, Average Number of
Products per Transaction for Loyalty without Recipe Transactions
The first row contains metrics for all stores
The Store ID and Region will be "O" for the first row
The report is emailed in an Excel file
The MAUR retains normal interactivity available in Excel files
