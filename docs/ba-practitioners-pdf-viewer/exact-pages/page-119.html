• Usage. Use case diagrams can be used to summarize the scope of a solution, highlighting the main
features to be added (i.e., the use cases). These diagrams also show the stakeholders who directly
interact with the solution (actors), and the interfaces that need to be created between the system features
(use cases) and the actors. Use case diagrams can also indicate use cases that are out of scope, which
is helpful to manage stakeholder expectations.
•
Relationship to requirements.
Use case diagrams help the project team plan and track progress in
building a solution. These diagrams help to summarize the scope of features and the relationship of
features to actors. Use case diagrams do not show requirements, but help to organize requirements for
business analysis efforts or layout in a requirements document.
