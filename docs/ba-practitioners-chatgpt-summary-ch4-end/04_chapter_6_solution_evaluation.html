# Chapter 6 - Solution Evaluation (Chương 6 - Đánh giá giải pháp)

Scope note: Chapter 6 focuses on whether the solution delivered the expected outcomes. It connects requirements, acceptance criteria, testing evidence, operational metrics, benefits, go/no-go decisions, signoff, and long-term performance improvement.

Learning map (Bản đồ học tập)

  • Solution evaluation focuses on whether value was achieved, not just whether work was completed.
  • Evaluation criteria should connect to business goals, KPIs, acceptance criteria, and expected benefits.
  • Evidence sources include tests, surveys, focus groups, operational metrics, and financial measures.
  • Go/no-go decisions need visibility into defects, readiness, residual risk, and business consequences.
  • Long-term performance evaluation supports improvement, replacement, or phase-out.

Use it when

How to apply

Connect each metric to a business objective, define expected and actual values, confirm data sources, and decide how variance will be interpreted.

Practical example

A support portal is expected to reduce ticket volume by 15%. The BA defines the pre-launch baseline, the measurement period, the data owner, and the threshold for acceptable variance before the release goes live.

Common mistakes / anti-patterns

  • Choosing metrics after launch when data is unavailable
  • Treating delivery metrics as proof of business value
  • Ignoring tolerance ranges and measurement timing

Checklist

  • Define expected value and actual data source
  • Confirm owner and measurement timing
  • Set thresholds or tolerance ranges
  • Decide how results will influence action
Metric familyWhat it can showExample
Business goalWhether the solution addresses the reason for investmentReduced cycle time, increased retention
CustomerWhether users or customers experience improvementSatisfaction score, complaint rate, adoption
OperationalWhether the organization performs betterThroughput, error rate, backlog age
FunctionalWhether expected behavior worksPassed UAT scenarios, acceptance criteria
FinancialWhether benefits justify costSavings, revenue, avoided cost, ROI
